# Wiki 人物健康检查工具 - 检查清单

## 功能完整性检查

### 名字合并检查
- [ ] 能检测拼写差异的名字（如 Zelensky vs Zelenskyy）
- [ ] 能检测中英文名对应（如 特朗普 vs Donald Trump）
- [ ] 能检测别名（如 Kanye West vs Ye）
- [ ] 能检测称谓差异（如 Ali Khamenei vs Ayatollah Ali Khamenei）
- [ ] LLM 分析基于新闻全文，不臆测
- [ ] 提供基于新闻原文的理由和证据
- [ ] 生成合并建议报告

### 时间线质量检查
- [ ] 能检测时间线是否只是新闻标题
- [ ] 能获取关联的新闻全文
- [ ] LLM 能生成人物行为总结
- [ ] 处理上下文限制，不截断新闻
- [ ] 生成时间线改进报告

### 新闻来源检查
- [ ] 能通过 feed_id 获取新闻源信息
- [ ] 能检测缺少来源的新闻条目
- [ ] 能补充新闻来源信息
- [ ] 更新后的 Wiki 页面包含来源字段

### 非人物实体检测
- [ ] 能检测国家名（如 Iran, Israel, Bahrain）
- [ ] 能检测组织名（如 Planet Labs PBC）
- [ ] LLM 能辅助判断实体类型
- [ ] 提供处理建议

## 命令行工具检查

### 基本命令
- [ ] `rss-news wiki health-check` 能正常运行
- [ ] `--check names` 只检查名字合并
- [ ] `--check timeline` 只检查时间线质量
- [ ] `--check source` 只检查新闻来源
- [ ] `--check all` 运行所有检查（默认）

### 高级功能
- [ ] `--fix` 能自动修复检测到的问题
- [ ] `--report` 能生成详细报告
- [ ] 输出格式清晰易读

## 验收标准

### 必须满足
1. 能检测并建议合并重复的人物名称
2. LLM 分析必须基于新闻全文，给出理由
3. 时间线改进能总结人物行为
4. 新闻来源信息完整
5. 命令行工具运行正常

### 应该满足
1. 非人物实体检测准确
2. 自动修复功能正常
3. 报告格式清晰
4. 性能可接受

### 可选满足
1. 支持导出报告为文件
2. 支持交互式合并确认
3. 支持增量检查
